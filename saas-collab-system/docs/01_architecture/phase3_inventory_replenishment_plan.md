# 阶段3库存预警与补货建议方案

## 输入与计算

建议模型使用可售库存、在途库存、日均销量、销售波动、安全库存、补货周期和采购交期。系统计算缺货预测、超储预测、建议补货量、建议补货日期、建议置信度和建议原因。

建议示例公式：

```text
需求覆盖量 = 日均销量 * (采购交期 + 补货周期 + 安全天数)
建议补货量 = max(0, 需求覆盖量 - 可售库存 - 在途库存)
```

销售波动、缺失数据、异常数据和数据质量状态应影响建议置信度，并在原因中说明。所有建议必须记录 tenant、商品、计算时间、口径版本、数据来源和人工处理状态。

## 硬性边界

- 补货建议只能产生建议。
- 不自动生成正式采购订单。
- 不自动通知真实供应商。
- 不自动触发 RPA。
- 最终采购必须由授权人员确认，并形成审计记录。
