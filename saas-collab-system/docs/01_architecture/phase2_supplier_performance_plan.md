# 阶段2供应商绩效方案

## 数据范围与隔离

所有绩效记录、统计查询和导出必须按 `tenant_id + supplier_id` 限定范围。供应商只能查看自己的绩效明细和汇总；内部用户只能在后端 permissions 与 data_scope 授权范围内查看供应商绩效汇总。

## 指标定义

| 指标 | 说明 |
|---|---|
| 交付及时率 | 在承诺交期内完成或出货的任务占比 |
| 逾期率 | 超过承诺交期仍未完成的任务占比 |
| 异常率 | 发生质量、数量、交期或物流异常的任务占比 |
| 反馈及时性 | 供应商在规定反馈窗口内完成回填的比例或时长 |
| 出货准确率 | 出货数量、箱数、重量等与确认数据一致的比例 |
| 供应商评分 | 根据已确认指标、权重和规则版本计算的可解释评分 |

## 计算与展示原则

指标应保留统计周期、数据来源、规则版本、样本数和质量标记。缺失、未确认或跨 tenant 数据不得参与评分。供应商端仅展示本人范围数据；内部汇总页面不得将供应商数据暴露给无权限用户。阶段2使用 Mock 或脱敏样例，不写入真实供应商、物流、银行或订单数据。
