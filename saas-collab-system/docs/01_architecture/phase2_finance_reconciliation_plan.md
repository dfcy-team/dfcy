# 阶段2财务对账增强方案

## 数据范围

阶段2定义以下脱敏数据模型和流程：平台账单、取款记录、银行到账记录和手工导入批次。真实银行接口、真实资金指令和真实生产流水不在范围内。

## 对账流程

1. 通过 Mock、脱敏样例或批准的导入模板生成账单和到账记录。
2. 执行格式、币种、日期、金额范围、重复记录和 tenant 归属校验。
3. 基于受控规则进行自动匹配，并记录匹配依据和置信度。
4. 无法匹配或金额、日期、币种异常的记录标记为差异，进入人工处理。
5. 财务人员确认、忽略或调整差异时必须形成审计日志。

## 权限与脱敏

财务接口必须独立授权，至少要求 internal 身份和财务权限码；普通 internal、external 和 rpa 用户不得访问。页面、日志、导出和测试数据须遵循最小展示与脱敏规则，掩码银行账号、交易参考号和个人信息，不记录真实账号或完整流水。

## 审计与边界

每个导入、自动匹配和人工处理操作应记录 actor、tenant、来源、批次、时间、动作、前后状态和差异原因。RPA 不得访问财务接口或数据。阶段2只实现模型、流程、Mock/沙箱和测试，不直接接入真实银行接口，也不执行真实资金自动操作或自动对账结算。
