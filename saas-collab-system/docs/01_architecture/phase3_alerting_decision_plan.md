# 阶段3经营预警与决策方案

## 预警类型

库存不足、库存超储、销量异常下降、商品滞销、对账差异、供应商逾期、RPA连续失败、页面签名变化、数据同步失败、数据长期未更新。

## 通用规则

每条预警定义预警级别、预警规则、触发条件、去重规则、静默时间、处理状态、负责人、处理记录、关闭条件和审计要求。建议级别为 P0/P1/P2 或业务高/中/低，但业务预警等级不等同于架构问题等级。

- 去重键至少包含 tenant、预警类型、对象和规则版本。
- 静默时间由 tenant 或系统级配置控制，过期后可重新通知。
- 处理状态至少包含 open、acknowledged、in_progress、resolved、ignored。
- 关闭需要记录负责人、处理说明、证据和关闭时间。

## 硬性边界

- 预警不直接触发真实平台操作。
- 预警不直接创建付款。
- 预警不直接改变商品状态。
- 高风险动作必须进入审批或人工确认。
