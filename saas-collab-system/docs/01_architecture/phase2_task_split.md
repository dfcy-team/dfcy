# 阶段2任务角色拆分

## 架构设计员

架构设计员必须先输出安全方案、接口契约和验收标准，并在每个开发任务开始前确认数据范围、权限矩阵、状态流转、审计要求及发布门禁。

| 任务编号 | 任务名称 | 交付重点 |
|---|---|---|
| P2-ARCH-001 | 阶段2范围与边界定义 | 范围、不做事项、阶段1边界、系统接口分区和风险边界 |
| P2-ARCH-002 | API数据接入安全方案 | 凭据托管、签名、限流、重试、幂等、webhook、灰度和回退 |
| P2-ARCH-003 | 商品状态机设计 | 多来源状态优先级、流转规则、人工确认和审计 |
| P2-ARCH-004 | 财务对账增强方案 | 脱敏模型、匹配与差异流程、独立授权和审计 |
| P2-ARCH-005 | 供应商绩效方案 | `tenant_id + supplier_id` 范围、指标字典和汇总权限 |
| P2-ARCH-006 | RPA稳定性增强方案 | 幂等、串行、重试、人工接管和审批边界 |
| P2-ARCH-007 | 阶段2验收标准 | 功能、安全、隔离、构建、测试和发布验收清单 |
| P2-ARCH-008 | 真实平台接入专项评审模板 | 凭据、网络、数据、审计、灰度、回退与书面批准模板 |

## 开发A

开发A后续负责后端框架、状态机、对账模型、供应商绩效统计和 CI；所有实现必须保持 tenant 隔离、独立财务授权、统一响应和审计边界。

| 任务编号 | 任务名称 | 交付重点 |
|---|---|---|
| P2-A-001 | API接入配置与密钥加密模型 | 非明文凭据引用、加密存储模型、轮换元数据和最小权限访问 |
| P2-A-002 | API同步任务框架 | 连接器契约、同步状态机、幂等、重试、限流、质量校验和审计 |
| P2-A-003 | 商品状态机后端模型 | 状态来源、规则建议、人工确认、tenant 过滤和审计记录 |
| P2-A-004 | 财务对账基础模型 | 脱敏导入、自动匹配、差异标记、独立授权和审计 |
| P2-A-005 | 供应商绩效后端统计 | `tenant_id + supplier_id` 过滤、指标计算和内部汇总权限 |
| P2-A-006 | RPA稳定性后端字段与任务状态增强 | 心跳、重试上限、串行、manual_required、截图元数据和审批校验 |
| P2-A-007 | CI/CD检查固化 | 后端测试、契约测试、构建、依赖扫描和密钥扫描门禁 |

## 开发B

开发B后续负责 API 同步状态页、商品状态看板、财务差异页、供应商绩效页、RPA 失败转人工页和前端构建质量。前端仅展示后端返回的权限与数据范围，不实现真实权限判断。

| 任务编号 | 任务名称 | 交付重点 |
|---|---|---|
| P2-B-001 | API同步状态页面 | 同步状态、错误、重试、数据质量和 Mock/API 切换展示 |
| P2-B-002 | 商品状态看板 | 状态来源、建议原因、人工确认入口和 loading/error/empty 状态 |
| P2-B-003 | 财务对账差异页面 | 脱敏差异展示和后端财务授权结果处理，不处理真实资金操作 |
| P2-B-004 | 供应商绩效页面 | 供应商自身绩效与授权内部汇总展示，不访问内部或财务越界接口 |
| P2-B-005 | RPA失败转人工页面 | manual_required、失败原因、最后成功步骤、截图/日志元数据展示 |
| P2-B-006 | 阶段2前端构建优化观察 | 构建结果、chunk warning、性能观察和不阻断项记录 |
| P2-B-007 | 真实平台接入前端风险提示与配置占位 | Mock/沙箱状态、风险提示和配置占位，不保存或展示真实凭据 |

## 协作与安全前置条件

1. 架构设计员完成相关安全方案、接口契约和验收标准后，开发A和开发B才能开始对应任务。
2. 开发A先完成后端接口、权限、tenant 隔离和测试契约；开发B再按已确认契约联调页面。
3. 任何真实平台接入前必须单独完成专项安全评审并取得书面批准。
4. 未获批准时，开发A和开发B只能使用 Mock、placeholder 或批准的沙箱；不得使用真实账号、Token、Cookie、Session 或真实业务数据。
5. 真实平台审批不自动授权自动改价、清仓、补货、上下架、刊登或财务自动操作；这些高风险动作仍需独立评审。
