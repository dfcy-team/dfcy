# 阶段2任务角色拆分

## 架构设计员

架构设计员须在开发开始前输出并评审安全方案、接口契约、数据范围、权限矩阵、状态流转、测试验收标准和发布门禁。重点包括真实平台专项安全评审、高风险自动化独立审批、tenant 隔离、供应商数据范围、财务独立授权与 RPA 接口边界。

## 开发A

开发A后续负责：

- 后端 API 数据接入框架、连接器契约、同步状态机、幂等与审计能力。
- 商品状态识别规则、人工确认接口和 tenant 隔离。
- 财务对账模型、差异处理流程和独立授权控制。
- 供应商绩效统计、`tenant_id + supplier_id` 查询范围和汇总权限。
- 后端测试、契约测试、CI 检查和安全扫描接入。

开发A不得在未获专项审批时接入真实平台、真实银行或真实高风险自动化。

## 开发B

开发B后续负责：

- API 同步状态页和错误/重试/数据质量展示。
- 商品状态看板与人工确认交互占位或经批准的后端联调。
- 财务差异页，且不得在前端实现真实财务权限判断。
- 供应商绩效页，且供应商视图不得访问内部或财务接口。
- RPA 失败转人工页，且不得模拟 RPA Agent Token 或调用 Agent 执行端点。
- 前端构建、Mock/API 切换、错误态和空态验证。

## 协作顺序

1. 架构设计员先确认安全方案、接口契约和验收标准。
2. 开发A完成后端框架、权限与测试契约。
3. 开发B按已确认契约实现 Mock/API 切换和页面联调。
4. 架构设计员执行范围、安全、接口一致性和验收复审。

任何真实平台接入前必须单独通过安全评审；没有书面批准，开发A和开发B只能使用 Mock、placeholder 或批准的沙箱。
