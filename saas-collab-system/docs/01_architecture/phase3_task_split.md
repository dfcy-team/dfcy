# 阶段3任务拆分

## 架构设计员

| 编号 | 任务 | 验收重点 |
|---|---|---|
| P3-ARCH-001 | 阶段3范围冻结与边界 | 禁止事项、权限、tenant、审计和执行边界明确 |
| P3-ARCH-002 | 指标口径与数据治理 | 指标字典、血缘、版本、质量规则和脱敏规则明确 |
| P3-ARCH-003 | BI与预警接口契约 | 只读/建议接口、数据范围、响应结构和状态定义明确 |
| P3-ARCH-004 | 库存与生命周期建议方案 | 建议与人工确认边界明确，不生成真实采购或业务状态 |
| P3-ARCH-005 | 系统配置中心安全方案 | 配置审批、审计、环境隔离和真实平台禁令明确 |
| P3-ARCH-006 | 阶段3验收与发布门禁 | 测试、安全、构建、数据质量和回退标准明确 |

架构设计员必须先冻结指标、数据口径、接口和验收标准，再允许开发任务启动。

## 开发A后续任务

- P3-A-001 指标事实表/汇总模型与 tenant 隔离。
- P3-A-002 BI只读查询、数据质量和指标计算任务框架。
- P3-A-003 库存预警、补货建议和生命周期复盘建议模型。
- P3-A-004 经营异常预警、配置审批和审计接口。
- P3-A-005 阶段3后端测试、指标校验与 CI 门禁。

开发A负责指标聚合、预警规则、补货建议、生命周期分析、配置中心和报表 API。

## 开发B后续任务

- P3-B-001 BI经营分析与管理决策看板。
- P3-B-002 库存预警和补货建议页面，仅展示/人工确认占位。
- P3-B-003 生命周期复盘和异常预警页面。
- P3-B-004 系统配置中心与数据质量页面。
- P3-B-005 阶段3前端测试、构建和可访问性观察。

开发B负责 BI 看板、预警页面、补货建议页面、生命周期复盘页面和配置页面。

任何真实平台接入、高风险自动化、采购订单创建或资金动作，均不属于上述任务；必须单独安全和业务审批。

开发A/B开始前必须等待本规划 PR 合并 main。真实平台接入不属于阶段3默认开发权限。
