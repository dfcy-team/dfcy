# 阶段3任务拆分

## 架构设计员

架构设计员必须先冻结指标、数据口径、接口和验收标准，再允许开发任务启动。

| 任务编号 | 任务名称 | 交付重点 |
|---|---|---|
| P3-ARCH-001 | 阶段3范围与边界冻结 | 范围、禁止事项、tenant、权限、审计和执行边界 |
| P3-ARCH-002 | BI指标目录与口径 | 指标编码、来源、公式、粒度、权限、质量和版本 |
| P3-ARCH-003 | 库存预警与补货建议方案 | 阈值、建议、置信度、人工确认和禁止自动采购 |
| P3-ARCH-004 | 商品生命周期复盘方案 | 阶段、复盘周期、建议、确认和审计 |
| P3-ARCH-005 | 经营预警与决策规则 | 规则、级别、去重、静默、处理和关闭 |
| P3-ARCH-006 | 配置中心方案 | 范围、版本、生效、审批、审计和回滚 |
| P3-ARCH-007 | 数据治理方案 | 口径、血缘、质量、脱敏、保留和导出审计 |
| P3-ARCH-008 | 阶段3接口契约 | 分区、角色、数据范围、响应、分页和状态 |
| P3-ARCH-009 | 阶段3验收标准 | 功能、隔离、质量、测试、安全和发布门禁 |
| P3-ARCH-010 | 真实平台专项安全评审计划 | 凭据、网络、回调、灰度、回退、紧急停止和书面批准 |

## 启动与安全约束

1. 开发A/B开始前必须等待本规划 PR 合并 main。
2. 开发任务必须基于已冻结的指标、数据口径、接口和验收标准。
3. 真实平台接入不属于阶段3默认开发权限。
4. 补货建议、生命周期建议、预警和 BI 只产生建议或展示，不自动创建采购订单、不执行资金动作、不直接改变商品状态，也不触发真实 RPA。
