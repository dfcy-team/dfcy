# 阶段3范围冻结

## 目标

阶段3建设 BI 经营分析、管理决策指标、库存预警和补货建议、商品生命周期复盘、经营异常预警、系统配置中心、指标口径与数据治理，并补齐阶段3测试、验收和发布标准。

## 阶段3范围

BI经营分析覆盖销售、库存、商品、采购供应商、财务和 RPA 指标的只读看板与钻取设计；库存预警覆盖缺货、超储和滞销风险；补货建议只计算建议数量、日期、置信度与原因；生命周期复盘覆盖观察、成长、稳定、滞销、清仓候选、停售和归档建议；配置中心覆盖阈值、规则和展示配置的审批、版本和审计。

## 与阶段2边界

阶段2提供 tenant 隔离、权限、同步框架、商品状态建议、财务对账基础、供应商绩效、RPA 稳定性和前后端联调基线。阶段3只读取或汇总这些可信数据，提供分析、建议、预警和配置管理，不改变阶段2的业务审批和执行边界。

## 允许范围

- 指标字典、数据血缘、刷新频率、质量规则、口径版本与审批记录。
- BI 看板、数据钻取、导出权限方案和经营异常预警展示。
- 库存阈值、补货建议、生命周期复盘和人工确认流程。
- 系统配置中心的 Mock/沙箱配置模型、审批与审计设计。
- 只读 API、建议 API、通知占位、测试夹具和发布门禁。

## 禁止范围

- 直接接入真实 BigSeller、Shopee、TikTok/TK、银行、支付或其他生产平台。
- 自动改价、清仓、补货、上下架、刊登、自动创建采购订单、自动确认财务对账、自动改变商品生命周期状态。
- BI 直接修改业务主数据；预警直接触发真实 RPA 操作。
- 提交真实账号、密码、Token、Cookie、Session、API Key/API Secret 或真实业务数据。

## 不变安全边界

- API 负责数据与分析服务，不负责页面操作；RPA 只做受控页面执行和结果回写，不做业务判断。
- 前端只展示后端返回的 roles、permissions、data_scope，不承担真实权限判断。
- 供应商只能访问自身 tenant 和 supplier_id 范围；财务数据保持独立授权。
- 所有指标、建议、预警、配置和查询必须体现 tenant 隔离与审计。

## 真实平台专项安全评审边界

真实平台、真实凭据、OAuth、webhook、API Key/API Secret、网络出口、回调、灰度、回退、紧急停止和人工接管均不属于阶段3默认开发权限。任何接入前必须独立完成专项安全评审并获得书面批准。
