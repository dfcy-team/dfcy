# 阶段3指标与数据治理方案

## 指标口径

| 领域 | 核心指标 | 最小口径 |
|---|---|---|
| 经营分析 | GMV、订单量、毛利率、退款率、库存周转 | tenant、平台、店铺、商品、日期和币种维度 |
| 库存预警 | 可售库存、在途库存、预计可售天数、安全库存缺口 | 仅产生预警和补货建议，不创建采购单 |
| 生命周期 | 新品、正常销售、滞销、清仓候选、停售候选、归档候选 | 只读复盘和建议，最终状态由后端授权确认 |
| 供应商绩效 | 及时率、逾期率、异常率、反馈及时性、出货准确率 | `tenant_id + supplier_id + 统计周期` |
| 财务分析 | 对账匹配率、差异率、异常数量、处理时长 | 财务独立授权，使用脱敏字段 |
| 异常预警 | 同步失败、库存缺口、生命周期异常、绩效异常、RPA人工接管 | 预警不直接执行业务或 RPA 动作 |

## 数据治理规则

1. 指标必须记录来源表、来源批次、计算时间、口径版本、质量结果和 tenant。
2. 未通过质量校验、跨 tenant、过期或缺失数据不得进入正式指标汇总。
3. 配置变更、阈值变更、指标口径变更和人工确认均需审计日志。
4. 经营看板只读；导出、财务查看和供应商视图均由后端权限控制。
5. 预警、补货建议、生命周期建议必须展示证据与原因，支持人工确认或忽略，不得自动执行。
