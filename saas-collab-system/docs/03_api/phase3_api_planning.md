# 阶段3 API 规划

## 路径分区

| 分区 | 用途 | 阶段3限制 |
|---|---|---|
| `/api/internal/bi/*` | 经营分析、指标和异常预警 | 只读或建议，按 tenant/data_scope 过滤 |
| `/api/internal/inventory/*` | 库存预警和补货建议 | 不创建正式采购订单 |
| `/api/internal/lifecycle/*` | 生命周期复盘与建议 | 不直接改变业务生命周期状态 |
| `/api/internal/settings/*` | 配置草稿、审批和审计 | 不存储真实平台凭据 |
| `/api/finance/*` | 财务分析和对账只读数据 | 仅独立财务权限用户可访问 |
| `/api/external/*` | 供应商自身的只读绩效/预警 | 不访问内部或财务数据 |

所有接口保持 `{ success, code, message, data }` 响应结构。未实现接口必须标记 pending 或 mock，不得伪造 connected。

## 建议与执行边界

- 补货、生命周期和异常接口只返回建议、证据、阈值、口径版本与人工确认状态。
- 不提供自动创建采购订单、自动改价、自动清仓、自动上下架、自动付款或自动 RPA 执行端点。
- 任何后续动作必须由独立业务流程、后端权限、审批凭证、审计与人工接管控制。

## 数据与安全要求

- 读取和汇总必须按 tenant 过滤；供应商额外按 supplier_id 过滤。
- 指标接口记录口径版本、数据质量、刷新时间和来源批次。
- 不返回原始凭据、完整银行信息、跨 tenant 数据或敏感财务原始字段。
- 真实平台、webhook、API Key/API Secret、OAuth、网络出口和回调均需单独专项安全评审。
