# 阶段2进入说明

## 阶段2进入条件

阶段1 R1 最终结论为 PASS，`main` 已包含阶段1收尾报告和发布说明，`v0.2-phase1-complete` 已作为阶段1基线。阶段2开发任务必须从最新 `main` 创建，并具备已批准的范围、责任人、接口契约、验收标准和测试计划。

## 阶段2禁止事项

- 禁止提交真实凭据、真实业务数据和真实生产环境配置。
- 禁止直接连接真实 BigSeller、Shopee、TK/TikTok、银行、支付或其他外部生产平台。
- 禁止真实自动改价、清仓、补货、上下架、刊登和财务自动资金操作。
- 禁止 RPA 直连数据层或访问 finance/internal finance/admin，禁止前端承担真实权限判断。

## 开发A/开发B开始前置条件

架构设计员须先完成安全方案、接口契约和验收标准。开发A开始后端框架、状态机、对账模型、绩效统计或 CI 前，应确认 tenant、权限、审计、失败处理和 Mock/沙箱方案。开发B开始同步状态页、状态看板、财务差异页、绩效页或 RPA 人工接管页前，应确认对应后端路径、Mock/API 状态和前端数据范围。

## 真实平台与高风险RPA

是否允许真实平台：不允许，需专项安全评审。

是否允许真实高风险 RPA：不允许，需专项评审。

任何审批通过前，系统仅使用 Mock、placeholder 或批准的沙箱；真实平台接入批准不自动等于高风险动作批准。

## 阶段2第一个开发任务建议

建议先执行 `P2-A-001 API 数据接入安全方案`：完成连接器接口契约、凭据托管设计、签名/幂等/限流/重试规则、Mock/沙箱测试用例和专项安全评审清单，再开始任何后端或前端联调任务。
