# 阶段3进入说明

## 进入条件

阶段2已以 `v0.3-phase2-complete` 完成收尾。阶段3开发分支必须从最新 main 创建，并具备已批准的任务、接口契约、指标口径、数据范围、验收标准和测试计划。

## 首个建议任务

优先执行 P3-ARCH-002 与 P3-ARCH-003：冻结指标字典、数据血缘、质量规则和 BI/预警 API 契约，再进入任何模型或页面实现。

## 禁止事项

不允许直接连接真实平台、真实银行或支付；不允许提交真实凭据；不允许自动执行改价、清仓、补货、上下架、采购订单、资金操作或生命周期状态变更。

## 发布门禁

阶段3每个工作包均须通过 tenant/权限、数据质量、安全扫描、测试、构建和审计验证。真实平台接入与高风险自动化必须另行专项安全评审，不因阶段3规划或开发而自动获得授权。
