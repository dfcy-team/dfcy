# 阶段2收尾说明

## 收尾摘要

阶段2最终审核结论为 PASS。开发A后端框架与开发B前端联调已进入 main，整体集成审核、测试、构建、CI、RPA JSON 与安全扫描均已完成。

## 保留边界

- API 负责数据接入和同步编排，不负责页面操作。
- RPA 只通过 `/api/rpa/*` 回写执行结果，不做业务判断，不直连数据层。
- 前端不承载真实权限判断；供应商、财务和 tenant 数据范围以后端为准。
- 未实现的 RPA 管理查询接口继续标记 pending/mock。

## 真实平台与高风险自动化

不允许直接接入真实平台，必须另行完成专项安全评审。

不允许启用真实自动改价、清仓、补货、上下架、刊登、财务自动对账、付款、转账或提现。任何高风险自动化均需独立审批、审计、人工接管和紧急停止方案。

## 阶段3准备项

1. 制定阶段3范围、架构和验收标准。
2. 补充前端组件/页面冒烟测试，并评估 Element Plus 按需导入。
3. 明确 npm allow-scripts 的依赖治理策略。
4. 如有真实平台需求，先执行专项安全评审与凭据/网络/回退方案审查。
