# SANDBOX-ARCH-002 精简生产发布门禁

## 1. 发布路径

```text
Local Sandbox PASS
  -> PR独立审核
  -> main全量CI PASS
  -> 不可变镜像与制品清单
  -> 发布审批
  -> 生产备份与迁移预检
  -> 同摘要部署
  -> 冒烟测试
  -> 观察或回滚
```

本路径不要求常驻独立 Sandbox 主机，但不允许从开发电脑或未合并分支直接向生产传输源码和构建目录。

## 2. 发布输入

- 受保护 `main` 的固定 Git SHA。
- 来源 PR、风险等级和独立审核记录。
- 后端、前端不可变镜像 digest。
- 数据库迁移清单及迁移摘要。
- 全量后端、前端和合同测试结果。
- Docker、安全、密钥和真实平台连接扫描结果。
- SBOM或锁定依赖清单。
- 数据库备份、部署步骤、停止条件和回滚步骤。

任一输入缺失时不得开始生产部署。

## 3. 生产主机规则

1. 生产主机只执行镜像拉取、digest校验、迁移和服务启动，不执行源码构建。
2. 生产应用与数据库只使用生产账号、卷、网络、TLS和密钥。
3. 开发人员本地Sandbox凭据和数据不得进入生产。
4. 生产数据库不得向开发电脑开放。
5. Production Adapter、真实平台或高风险动作保持默认禁用，除非专项评审单独批准。
6. 数据库迁移失败、摘要不一致或配置检查失败必须立即停止。

## 4. 快速通道

L1低风险变更可省略单独架构复审报告，但仍必须经过PR、CI、独立审核和不可变制品发布。L2按模块合同执行。L3不得使用快速通道。

紧急修复不得跳过密钥扫描、独立审核、固定制品、数据库保护和回滚；允许缩短观察窗口，但必须记录原因和批准人。

## 5. 部署后验证

- 服务和依赖健康检查。
- 登录、刷新、退出及权限拒绝。
- 本次变更涉及的只读或低风险冒烟流程。
- tenant、财务、external和RPA边界不得漂移。
- migration状态、Celery、Redis和数据库连接正常。
- 日志不包含密码、Token、Cookie、Session或完整敏感字段。

冒烟失败、错误率异常、任务积压、数据库异常或安全边界漂移时立即回滚。

## 6. 仍需专项评审的事项

- 真实 BigSeller、Shopee、TikTok/TK 或其他平台接入。
- 真实RPA浏览器自动化。
- 自动采购、改价、清仓、停售、归档或上架。
- 银行、支付、付款、转账、提现和自动财务确认。
- 生产凭据托管、轮换、网络白名单及公网暴露。

本门禁通过不自动批准上述能力。
