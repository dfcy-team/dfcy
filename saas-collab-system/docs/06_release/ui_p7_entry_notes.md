## 允许事项

- API治理目录、合同详情和 Mock 一致性检查。
- 智能体能力声明与 Mock 评估占位。
- 双主机拓扑、试点就绪、备份恢复计划、发布回滚计划和容量观测。
- 受控试点的真实登录、后端 API、网络、恢复和浏览器 E2E 验证。
- 计划、审批、结果、证据和审计记录；不由 Web 页面直接执行主机命令。

## 禁止事项

- 不接入真实 BigSeller、Shopee、TikTok/TK、AI provider、银行或支付平台。
- 不提交真实 `.env`、账号、密码、Token、Cookie、Session、API Key、API Secret、私钥或备份密钥。
- 不开放 Web shell、Docker socket、数据库管理端口或直接部署/恢复/回滚 API。
- 不自动采购、改库存、刊登、改价、清仓、停售、归档、付款、转账或提现。
- 不让智能体、预警、治理检查或容量阈值触发真实 RPA。

## 建议实施顺序

1. API治理目录和精确权限/data_scope。
2. 智能体只读占位和 Mock 评估边界。
3. 试点就绪及双主机脱敏拓扑。
4. 备份恢复计划、演练记录和不可变审计。
5. 灰度发布、停止条件、回滚计划和容量观测。
6. 自动化、双主机、浏览器 E2E、安全和恢复验证。
7. 独立实现复审与 G5 发布审核。
