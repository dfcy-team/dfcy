# UI-P6 API接入与分析复盘进入说明

## 进入条件

- UI-P5 已合并最新 `main`。
- UI-P6 页面、API、权限和验收合同已冻结。
- 实施分支从最新 `main` 创建。
- 架构合同前审无未关闭 P0/P1。

## 允许实施

- 经营总览、销售归因、库存分析和角色工作台增强。
- 财务只读复盘和脱敏展示。
- 生命周期建议、人工确认提示和审计展示。
- 清仓审批查询与 Mock 申请占位。
- API配置、Mock同步、运行结果、数据质量和平台准入视图。
- 报表导出权限、脱敏和审计交互。
- 前后端权限、tenant、data_scope、状态和异常测试。

## 禁止事项

- 不接入真实平台、真实银行或支付系统。
- 不提交真实账号、密码、Token、Cookie、Session、API Key 或 API Secret。
- 不自动创建采购订单，不自动改库存、改价、刊登或通知真实供应商。
- 不自动清仓、停售、归档，不由预警或看板触发真实 RPA。
- 不执行付款、转账、提现或自动确认对账。
- 不把未验证能力标记为 connected。

## 建议实施顺序

1. 经营总览与数据质量组件。
2. 销售归因和库存钻取。
3. 财务只读复盘。
4. 生命周期与清仓申请占位。
5. API接入中心、同步运行和平台准入。
6. 报表导出审计联动。
7. 定向测试、全量测试、构建、浏览器验收和独立架构复审。

## 分支建议

合同前审通过后，在当前规划分支后续实施，或从其合并后的最新 `main` 创建 `feature/ui-p6-api-analysis-review`。不得从 UI-P5 旧功能分支继续开发。

## 发布边界

UI-P6 通过只表示本项目后端、Mock/Sandbox 和界面合同可用，不代表允许生产平台接入。真实平台接入、生产凭据托管和任何高风险自动化仍需专项安全评审。
