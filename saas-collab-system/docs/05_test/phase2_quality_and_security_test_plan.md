# 阶段2质量与安全测试计划

## 1. 测试目标

阶段2以 Mock、沙箱和脱敏样例验证数据接入、权限隔离、供应商绩效、财务边界和 RPA 稳定性。不执行真实平台调用、真实财务对账或真实高风险自动化。

## 2. 必测工作包

| 领域 | 必测内容 | 通过标准 |
|---|---|---|
| API 数据接入 | 契约校验、幂等、限流、超时、重试、数据质量 | 可重放且不重复写入；失败可审计 |
| 商品状态识别 | 规则优先级、人工复核、tenant 隔离 | 不跨 tenant 读取或更新；人工复核可追踪 |
| 财务 | 独立权限、导入校验、异常处理、审计 | 普通 internal、external、rpa 均被拒绝 |
| 供应商绩效 | tenant_id + supplier_id 数据范围 | 供应商只能查询并操作自身数据 |
| RPA 稳定性 | 心跳、幂等、串行、重试、manual_required、截图元数据 | complete/fail 只回写执行结果，不改变业务决策 |
| 前端 | Mock/API 切换、loading/error/empty、统一响应处理 | pending 不标为 connected；无 `/admin/` 调用 |
| CI/CD | 后端测试、前端构建、依赖与密钥扫描 | 非生产配置下可复现，失败阻断发布 |

## 3. 安全回归清单

- 扫描真实 `.env`、私钥、证书、Token、Cookie、Session、API Key 和数据库密码。
- 验证 RPA 不访问 `/api/finance/*`、`/api/internal/finance/*`、`/admin/`，也不直连数据层。
- 验证供应商页面和接口不访问 `/api/internal/*` 或财务数据。
- 验证前端不硬编码真实公网 API、账号或 Token。
- 验证日志、截图、错误响应和测试夹具均使用 demo/placeholder 数据。

## 4. 证据要求

每个工作包应提交可复现命令、执行日期、环境说明、成功/失败结果和已知限制。未执行项必须标记为“未执行”并说明原因，不得以构建产物或口头描述替代测试证据。
