# SANDBOX-ARCH-002 精简验收清单

## 1. 所有变更必检

- [ ] 变更已登记模块、负责人和风险等级。
- [ ] 变更文件位于模块允许范围；公共合同变化已标明消费者。
- [ ] 本地 Sandbox 只使用合成数据和示例凭据。
- [ ] 模块单元测试通过。
- [ ] tenant、权限和data_scope相关测试按风险执行。
- [ ] API成功、分页及400/401/403/404/409/422合同未被破坏。
- [ ] 不存在真实平台连接、生产数据或真实凭据。
- [ ] PR由至少一名非作者审核。
- [ ] 必需 GitHub CI全部成功，审核HEAD未变化。

## 2. L1门禁

- [ ] 不修改数据库模型、迁移、权限、data_scope、状态机或公共API合同。
- [ ] 页面至少覆盖loading、empty、error或本次涉及状态。
- [ ] 前端构建和涉及组件测试通过。
- [ ] 不存在接口却标记为connected。

满足以上项目即可合并，不要求固定R2/R3。

## 3. L2门禁

- [ ] 模块合同简表完整。
- [ ] 普通增删改的权限和tenant隔离测试通过。
- [ ] 数据库迁移一致性检查通过。
- [ ] 状态字段不能通过通用PATCH绕过合法动作。
- [ ] 上游/下游Mock与真实本地接口字段一致。
- [ ] 前后端模块测试、构建和合同测试通过。

## 4. L3门禁

- [ ] 完整API、权限、data_scope、状态机和审计合同已冻结。
- [ ] 跨tenant、跨角色、跨供应商和财务越权测试通过。
- [ ] 模型直接写入、bulk、并发、重复操作和状态冲突防绕过测试通过。
- [ ] 财务字段脱敏、导出权限和审计通过。
- [ ] RPA、真实平台、密钥和高风险自动化保持禁用。
- [ ] 全量后端、前端、迁移、Docker和安全检查通过。
- [ ] 架构审核、发布审批和回滚验证完成。
- [ ] 如发现P0/P1，已完成定向整改和独立复审。

## 5. main与发布前全量门禁

- [ ] `python manage.py check --deploy`通过。
- [ ] `python manage.py makemigrations --check --dry-run`无差异。
- [ ] 全量pytest通过。
- [ ] `npm ci`、`npm test`、`npm run build`通过。
- [ ] Docker Compose配置通过。
- [ ] API路径、connected状态、禁止文件和密钥扫描通过。
- [ ] CI从受保护main构建不可变镜像并记录Git SHA和digest。
- [ ] SBOM、迁移摘要、测试结果和来源PR可追溯。

## 6. 生产部署门禁

- [ ] 只部署CI批准的同一镜像digest，目标主机不构建。
- [ ] `.env`、TLS私钥和生产凭据未进入Git、命令输出或日志。
- [ ] 数据库备份完成且可读取，迁移和回滚步骤获批。
- [ ] 发布窗口、负责人、停止条件和回滚负责人明确。
- [ ] 部署后健康检查、登录和只读冒烟测试通过。
- [ ] 观察期内错误率、任务队列、数据库和资源指标正常。

存在P0必须停止；存在未关闭P1不得发布。P2必须登记责任人和处理期限。
